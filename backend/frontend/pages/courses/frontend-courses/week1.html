1. Internet vs World Wide Web

The Internet is a massive global network of computers connected using cables, satellites, and wireless signals.

Internet vs World Wide Web (WWW)

The Internet is a global network of interconnected computer networks that allows computers, servers, and devices worldwide to communicate and share data.

The Internet acts as an infrastructure made up of hardware like routers, servers, cables, and software protocols such as TCP/IP. It provides the basic foundation for data transfer.

The Internet itself does not provide information. Instead, it acts as a medium that enables multiple services such as email, file transfer, video conferencing, cloud computing, online gaming, and the World Wide Web.

The World Wide Web (WWW) is a service that runs on the Internet. It is an information system consisting of interlinked web pages and websites that users access through web browsers.

The World Wide Web uses technologies such as HTML for creating web pages, URLs for identifying resources, HTTP and HTTPS for communication, and web browsers to display content.

The Internet can exist without the World Wide Web, but the World Wide Web cannot exist without the Internet. This means WWW is a subset of the Internet.

The Internet focuses on data communication between devices, whereas the World Wide Web focuses on accessing and sharing information in the form of web pages.

The Internet uses TCP/IP protocols to transfer data packets between devices, while the World Wide Web mainly uses HTTP and HTTPS protocols to access web content.

An easy way to understand the difference is by analogy: the Internet is like a road system, and the World Wide Web is like vehicles traveling on those roads.

The Internet was developed earlier, originating from ARPANET in the 1960s, while the World Wide Web was invented later in 1989 to simplify information sharing.

In summary, the Internet is a global communication network, whereas the World Wide Web is a collection of interlinked documents accessed using the Internet.

The World Wide Web (WWW) is a service that runs on top of the internet. It allows users to access documents (web pages) using browsers.

  • Internet = Network infrastructure
  • Web = Websites, web apps, URLs
💡 Example:
Email, WhatsApp → Internet Google, Facebook → Web + Internet

2. How the Web Works (Request–Response)

When you type a website URL:

How the Web Works

The World Wide Web works on a client-server model where a user requests information and a server responds by providing that information.

When a user enters a website address (URL) into a web browser, the browser acts as a client and sends a request to a web server.

The browser first checks the Domain Name System (DNS) to convert the domain name into an IP address so that the correct server can be located.

After obtaining the IP address, the browser sends an HTTP or HTTPS request to the web server asking for the required web page.

The web server receives the request, processes it, and locates the requested resources such as HTML files, CSS files, JavaScript files, images, or data.

If the request is valid, the server sends an HTTP response back to the browser containing the requested web page data.

The data is transferred over the Internet in small packets using TCP/IP protocols to ensure reliable communication.

Once the browser receives the response, it starts rendering the web page by interpreting HTML for structure, CSS for styling, and JavaScript for interactivity.

The browser then displays the fully loaded web page on the user’s screen, allowing interaction such as clicking links or submitting forms.

If the web page contains links, images, or scripts from other servers, the browser sends additional requests to load those resources.

This entire process usually happens within a few seconds, making the web appear fast and seamless to users.

In summary, the web works by sending requests from a client to a server, processing those requests, and delivering web content back to the client through the Internet.

  1. Browser sends request (HTTP)
  2. Server processes request
  3. Server sends response (HTML/CSS/JS)
  4. Browser renders the page
🔑 Key Term: HTTP = HyperText Transfer Protocol

3. How Browsers Work Internally

How a Browser Works Internally

A web browser is a software application that allows users to access, retrieve, and display web content from the Internet.

When a user enters a URL into the browser, the browser begins by breaking the URL into components such as protocol, domain name, and resource path.

The browser contacts the Domain Name System (DNS) to translate the domain name into an IP address of the web server.

After obtaining the IP address, the browser establishes a connection with the web server using TCP and, in the case of HTTPS, performs a secure TLS handshake.

The browser then sends an HTTP or HTTPS request to the server asking for the required web resources.

Once the server responds, the browser receives the response data, which usually includes HTML, CSS, JavaScript, images, and other assets.

The browser starts by parsing the HTML document to create a Document Object Model (DOM), which represents the structure of the web page.

At the same time, CSS files are parsed to build the CSS Object Model (CSSOM), which defines the visual styling of the page.

The browser combines the DOM and CSSOM to form a render tree, which contains only the elements that need to be displayed on the screen.

The layout process then calculates the size and position of each element in the render tree based on the viewport size.

After layout calculation, the browser paints pixels on the screen and displays the visible content to the user.

JavaScript is executed by the browser’s JavaScript engine, which can modify the DOM and CSSOM dynamically.

Whenever changes occur due to JavaScript or user interaction, the browser may reflow and repaint parts of the page.

In summary, a browser works internally by fetching web resources, parsing and processing them, building render structures, and finally displaying interactive content to the user.

  • HTML → DOM (Document Object Model)
  • CSS → CSSOM
  • DOM + CSSOM → Render Tree
  • JS → Interactivity

Browsers like Chrome use engines:

  • Blink → Rendering
  • V8 → JavaScript execution
📌 Interview Line:
“Browser parses HTML, builds DOM, applies CSS, then executes JavaScript.”

4. Client vs Server Architecture

Client vs Server Architecture

Client–Server Architecture is a computing model where tasks and responsibilities are divided between two main components: the client and the server.

The client is the user-facing part of an application. In web development, the client is usually a web browser that displays the user interface and handles user interactions.

Frontend technologies such as HTML, CSS, and JavaScript run on the client side and are responsible for layout, design, animations, and user actions.

The server is the backend part of the system that processes requests, applies business logic, accesses databases, and sends responses back to the client.

When a user performs an action on the frontend, such as clicking a button or submitting a form, the client sends a request to the server.

This request is usually sent using HTTP or HTTPS and contains information like form data, API parameters, or authentication tokens.

The server receives the request, processes it using backend logic, and may interact with a database to fetch or store data.

After processing, the server sends a response back to the client, usually in the form of HTML, JSON, or other data formats.

The client then uses this response to update the user interface dynamically without reloading the entire page.

In frontend development, the client focuses on user experience and presentation, while the server focuses on data processing and security.

This separation of responsibilities makes applications scalable, maintainable, and easier to develop.

A simple way to understand this is: the client asks for data, the server provides the data, and the client displays it to the user.

In summary, Client–Server Architecture allows frontend and backend to work together efficiently by clearly dividing user interaction and data processing.

Client (Frontend)

  • Browser
  • HTML, CSS, JS
  • User interface

Server (Backend)

  • Node.js / Java / Python
  • Business logic
  • Database

Example: Login form → Client Password verification → Server

5. Markup Languages Explained

Markup Language

A markup language is a system used to structure, format, and describe the content of a document using predefined tags.

In frontend development, markup languages are mainly used to define the structure and meaning of web content.

Markup languages use tags enclosed in angle brackets to mark different parts of content such as headings, paragraphs, images, and links.

HTML is the most commonly used markup language in frontend development and forms the backbone of every website.

Markup languages do not perform logic or calculations; they only describe how content is organized.

In a frontend course, markup languages are taught first because they provide the basic structure on which styling and interactivity are built.

Each tag in a markup language has a specific purpose, such as defining text, grouping elements, or embedding media.

Markup languages help browsers understand which content is important and how it should be displayed.

Markup languages are different from programming languages because they do not include conditions, loops, or functions.

In web development, markup languages work together with CSS and JavaScript to create complete user interfaces.

HTML structures the content, CSS styles the content, and JavaScript adds behavior and interactivity.

Using proper markup improves accessibility, search engine optimization, and maintainability of web pages.

In summary, a markup language defines the structure and meaning of content, making it readable for both browsers and developers.

  • HTML – Page structure
  • XML – Data storage & transfer
  • XHTML – Strict HTML rules
