1. What is CSS?

CSS stands for Cascading Style Sheets. It is a style sheet language used to control the visual presentation of HTML documents. While HTML is responsible for creating the structure of a webpage—such as headings, paragraphs, images, links, and tables—CSS is responsible for defining how those elements appear on the screen. In simple terms, HTML builds the structure of a house, and CSS decorates and designs that house by adding colors, layouts, spacing, and visual effects. The primary purpose of CSS is to separate content from design. This separation improves maintainability, readability, and scalability of web applications. Instead of writing styling code repeatedly inside HTML elements, developers can define styles in a separate CSS file and apply them across multiple pages. This makes websites easier to manage and update. CSS syntax consists of three main components: the selector, the property, and the value. The selector specifies which HTML element(s) the style will be applied to. The property defines what aspect of the element will be changed, such as color, font-size, margin, or background-color. The value specifies the setting for that property. For example: p { color: blue; } In this rule, "p" is the selector, "color" is the property, and "blue" is the value. There are three main ways to apply CSS to an HTML document. Inline CSS is written directly inside an HTML tag using the style attribute. Internal CSS is written inside a style tag within the head section of the HTML document. External CSS is written in a separate .css file and linked to the HTML document using the link tag. In professional web development, external CSS is the most recommended approach because it promotes clean code and reusability. The word “Cascading” in CSS refers to the priority system used when multiple style rules apply to the same element. The browser determines which rule to apply based on specificity, importance, and source order. Inline styles generally have higher priority than internal styles, and internal styles have higher priority than external styles. Specificity calculations and inheritance rules also play an important role in determining the final computed style of an element. Modern CSS provides advanced layout systems such as Flexbox and CSS Grid, which allow developers to create responsive and complex layouts efficiently. CSS also supports animations, transitions, media queries for responsive design, and custom properties (variables), making it a powerful tool for building professional, scalable, and visually appealing websites.

CSS (Cascading Style Sheets) controls the visual appearance of web pages. 

h1 {
  color: blue;
  font-size: 24px;
}
CSS separates structure (HTML) from design.

2. CSS Selectors

CSS Selectors are patterns used to target and select HTML elements so that styles can be applied to them. In CSS, selectors play a fundamental role because without selectors, the browser would not know which elements should receive specific styling rules. A selector can target elements based on their tag name, class, id, attributes, relationship with other elements, or even their state (such as hover or focus). Understanding selectors is essential for writing efficient, maintainable, and scalable CSS. The most basic type of selector is the Element Selector. It targets all elements of a specific HTML tag. For example, the selector p will apply styles to all paragraph elements in the document. This type of selector is simple but not always precise if you want to style only specific elements. The ID Selector is used to target a single unique element. It is defined using the hash (#) symbol followed by the id name. For example, #header selects the element with id="header". Since IDs must be unique within a page, this selector has high specificity and should be used carefully to avoid conflicts. The Class Selector is one of the most commonly used selectors in modern web development. It is defined using a dot (.) followed by the class name. For example, .container selects all elements with class="container". Unlike IDs, classes can be reused on multiple elements, making them ideal for reusable styling patterns. The Universal Selector is represented by an asterisk (*). It selects all elements in the document. Although powerful, it should be used cautiously because it can impact performance if applied broadly. The Grouping Selector allows multiple selectors to share the same style rules. For example, h1, h2, h3 { color: blue; } applies the same style to multiple heading levels. This improves code efficiency and reduces repetition. The Descendant Selector targets elements that are inside another element. For example, div p selects all paragraph elements inside a div. This is useful for contextual styling. The Child Selector (>) targets only direct children of an element. For example, div > p selects only those paragraphs that are directly inside a div, not nested deeper. The Attribute Selector targets elements based on the presence or value of attributes. For example, input[type="text"] selects only text input fields. This is especially useful in form styling. Pseudo-class Selectors target elements based on their state. Examples include :hover, :focus, :active, and :nth-child(). These are commonly used for interactive effects and dynamic behavior without JavaScript. Pseudo-element Selectors allow styling specific parts of an element. Examples include ::before, ::after, ::first-letter, and ::first-line. These are used to insert decorative content or style specific portions of text. In addition to understanding types of selectors, it is important to understand Specificity. Specificity determines which selector takes precedence when multiple rules target the same element. ID selectors have higher specificity than class selectors, and class selectors have higher specificity than element selectors. Efficient use of selectors improves code readability, performance, and maintainability. In professional development, developers prefer class-based styling combined with proper structure and naming conventions to maintain scalable and modular CSS architecture.

  • Element selector → p {}
  • Class selector → .box {}
  • ID selector → #header {}
  • Descendant selector → div p {}
.box {
  background: #fbbf24;
  padding: 20px;
}

3. Box Model

CSS Box Model

The CSS Box Model is a fundamental concept in front-end development. Every HTML element is rendered as a rectangular box, and understanding how this box behaves is essential for creating structured and responsive layouts.

The total size of an element is calculated using four primary layers: Content, Padding, Border, and Margin. These layers together determine spacing, alignment, and layout behavior.

1. Content Area

The content area is the innermost part of the box where text, images, or other media are displayed. By default, width and height properties apply only to this area.

2. Padding

Padding creates internal spacing between the content and the border. It improves readability and provides visual breathing space inside elements.

3. Border

The border surrounds the padding and content. It defines the visible boundary of an element and can be customized using width, style, and color properties.

4. Margin

Margin is the outermost layer of the box. It creates space between elements and ensures proper layout structure.

Total Width = Content + Left/Right Padding + Left/Right Border + Left/Right Margin

Total Height = Content + Top/Bottom Padding + Top/Bottom Border + Top/Bottom Margin

Every element in CSS follows the Box Model:

  • Content
  • Padding
  • Border
  • Margin

4. Flexbox (Modern Layout)

Flexbox, also known as the Flexible Box Layout, is a modern CSS layout model designed to create responsive and efficient layouts. It allows developers to align and distribute space among items within a container even when their sizes are dynamic or unknown.

Unlike traditional layout methods such as floats or positioning, Flexbox provides better control over alignment, direction, and spacing. It works in one dimension at a time — either horizontally (row) or vertically (column).

1. Flex Container

A Flex Container is the parent element that holds flex items. To activate Flexbox, you apply display: flex; to the parent element. Once enabled, all direct child elements automatically become flex items.

2. Flex Items

Flex items are the direct children of a flex container. They can grow, shrink, or align themselves based on the container’s properties. Flex items automatically adjust their size to fit available space.

3. Main Axis and Cross Axis

Flexbox operates using two axes: Main Axis (primary direction defined by flex-direction) and Cross Axis (perpendicular to the main axis). Alignment and spacing are controlled based on these axes.

4. Important Flex Container Properties

Some commonly used flex container properties include:

  • flex-direction – Defines row, column, row-reverse, or column-reverse
  • justify-content – Aligns items along the main axis
  • align-items – Aligns items along the cross axis
  • flex-wrap – Allows items to wrap onto multiple lines
  • align-content – Controls spacing between multiple rows

5. Important Flex Item Properties

Flex items have their own properties for controlling size and alignment:

  • flex-grow – Defines how much the item should grow
  • flex-shrink – Defines how much the item should shrink
  • flex-basis – Sets the initial size of the item
  • align-self – Overrides alignment for a single item

display: flex; → Activates Flexbox on a container

Flexbox automatically manages spacing, alignment, and item sizing to create responsive and flexible layouts. 

.container {
  display: flex;
  justify-content: center;
  align-items: center;
}
  • justify-content
  • align-items
  • flex-direction

Flexbox is used for navigation bars, cards, alignment.

5. CSS Grid

CSS Grid is a powerful two-dimensional layout system designed to handle both rows and columns simultaneously. It provides precise control over complex web layouts and makes it easier to build responsive designs without relying on floats or complicated positioning techniques.

Unlike Flexbox, which works in one dimension (row or column at a time), CSS Grid allows developers to control layout in both horizontal and vertical directions at the same time.

1. Grid Container

A Grid Container is the parent element that holds grid items. To activate Grid layout, you apply display: grid; to the parent element. All direct child elements automatically become grid items.

2. Grid Items

Grid items are the direct children of the grid container. They can be positioned precisely using row and column lines, and they can span across multiple rows or columns if required.

3. Grid Lines

Grid lines are the dividing lines that make up the structure of the grid. Horizontal lines define rows, and vertical lines define columns. These lines help position grid items accurately within the layout.

4. Important Grid Container Properties

Commonly used properties for grid containers include:

  • grid-template-columns – Defines the number and size of columns
  • grid-template-rows – Defines the number and size of rows
  • gap – Adds space between rows and columns
  • justify-items – Aligns items horizontally inside cells
  • align-items – Aligns items vertically inside cells

5. Important Grid Item Properties

Grid items can control their placement using:

  • grid-column – Defines column start and end position
  • grid-row – Defines row start and end position
  • grid-area – Assigns a named area for layout

6. Grid Template Areas

The grid-template-areas property allows developers to create visual layout structures by naming specific areas of the grid. This makes complex layouts easier to read and maintain.

display: grid; → Activates CSS Grid on a container

CSS Grid provides complete control over both rows and columns, making it ideal for building complex and structured page layouts. 

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 20px;
}

Grid is powerful for full-page layouts.

6. Responsive Design

Responsive Design is an approach to web development that ensures a website looks and functions properly on all screen sizes and devices, including desktops, tablets, and mobile phones. It focuses on creating flexible layouts that automatically adjust according to the user's screen resolution.

Instead of creating separate websites for different devices, responsive design uses fluid grids, flexible images, and media queries to adapt the layout dynamically. This improves user experience and accessibility.

1. Fluid Grid Layout

A fluid grid uses relative units like percentages (%) instead of fixed units like pixels (px). This allows elements to resize proportionally based on the screen width.

2. Flexible Images

Images in responsive design automatically scale within their parent container. This is usually achieved using properties like max-width: 100%; so images do not overflow.

3. Media Queries

Media queries allow developers to apply different CSS styles based on screen size, resolution, or device type. They help customize layout behavior for mobile, tablet, and desktop screens.

  • @media (max-width: 768px) – Styles for tablets and small screens
  • @media (max-width: 480px) – Styles for mobile devices

4. Mobile-First Approach

The mobile-first approach means designing for smaller screens first, then gradually adding styles for larger screens. This ensures better performance and improved usability on mobile devices.

5. Responsive Typography

Responsive typography uses relative units such as em, rem, or vw so that text scales properly on different screen sizes.

Responsive Design = Flexible Layout + Media Queries + Relative Units

It ensures consistency, usability, and accessibility across all devices. 

@media (max-width: 768px) {
  body {
    background: black;
  }
}

Media queries allow websites to adapt to different screen sizes.

7. Animations & Transitions

Animations and Transitions in CSS are used to create visual effects that improve user experience and make web interfaces more interactive. They allow elements to change smoothly from one style to another instead of changing instantly.

Transitions are used for simple state changes, such as hover effects, while animations provide more complex control over movement, timing, and multiple stages of transformation.

1. CSS Transitions

Transitions allow property changes to happen smoothly over a specified duration. They are commonly used for hover effects, button interactions, and color changes.

  • transition-property – Specifies the CSS property to animate
  • transition-duration – Defines how long the effect lasts
  • transition-timing-function – Controls animation speed curve (ease, linear, ease-in-out)
  • transition-delay – Adds delay before the animation starts

2. CSS Animations

Animations allow more advanced control compared to transitions. They use keyframes to define multiple stages of movement or style changes.

  • @keyframes – Defines animation steps
  • animation-name – Specifies the animation to use
  • animation-duration – Sets how long the animation runs
  • animation-iteration-count – Controls how many times it repeats
  • animation-direction – Defines normal, reverse, or alternate motion

3. Timing Functions

Timing functions control the acceleration and deceleration of animations. Common values include:

  • ease – Starts slow, speeds up, then slows down
  • linear – Constant speed
  • ease-in – Starts slow
  • ease-out – Ends slow
  • ease-in-out – Slow start and slow end

4. Transform with Animation

Animations often use the transform property to create movement effects such as translate, rotate, scale, and skew. These transformations enhance visual engagement without affecting document flow.

Transition = Smooth change between two states

Animation = Multi-step movement defined using keyframes

Animations and transitions improve interactivity and create modern UI experiences. 

button {
  transition: all 0.3s ease;
}

button:hover {
  transform: scale(1.1);
}

Animations improve user experience.
