1. What is DOM?

DOM stands for Document Object Model. It is a programming interface for web documents that represents the structure of an HTML or XML page as a tree of objects. The DOM allows programming languages like JavaScript to access, modify, and manipulate the content, structure, and styles of a webpage dynamically.

When a web page loads, the browser creates a DOM of the page. Every element, attribute, and piece of text becomes a node in this tree structure. Developers can interact with these nodes using JavaScript.

1. DOM Structure

The DOM represents the document as a hierarchical tree structure. The top of the tree is the document object, followed by the html element, then head and body, and so on.

  • Document → Root of the webpage
  • Element Nodes → HTML tags like div, p, h1
  • Text Nodes → The actual text inside elements
  • Attribute Nodes → Attributes like class, id, src

2. Why DOM is Important

The DOM allows developers to:

  • Access HTML elements
  • Change content dynamically
  • Modify CSS styles
  • Add or remove elements
  • Handle events like clicks and form submissions

3. DOM Manipulation

Using JavaScript, developers can select elements using methods such as:

  • getElementById()
  • getElementsByClassName()
  • querySelector()
  • querySelectorAll()

After selecting elements, their content, attributes, and styles can be modified.

4. DOM Events

The DOM supports event handling, which allows interaction between the user and the webpage. Common events include:

  • click
  • mouseover
  • keydown
  • submit

DOM = Structured representation of a webpage + JavaScript interaction

It enables dynamic, interactive, and responsive web applications.

DOM stands for Document Object Model. When a browser loads HTML, it converts it into a tree structure called the DOM.

HTML → Parsed by browser → Converted into DOM Tree

Each HTML element becomes a JavaScript object. This allows us to manipulate elements dynamically. 

document.body.style.background = "black";

2. Selecting Elements (Deep Dive)

Before JavaScript can modify anything on a webpage, it must first select that element from the DOM. Selecting elements is the foundation of DOM manipulation.

Think of it like this: If HTML is a house, selecting elements is choosing which room you want to enter.

🔹 Why Selecting Elements is Important?

  • Change text content
  • Modify styles dynamically
  • Add animations
  • Attach event listeners
  • Validate forms

1️⃣ getElementById()

Used to select a single element using its unique id. IDs must be unique inside an HTML document. 

const heading = document.getElementById("title");
heading.style.color = "blue";

✔ Fast ✔ Direct ✔ Best for unique elements

2️⃣ getElementsByClassName()

Used to select multiple elements that share the same class. It returns an HTMLCollection. 

const items = document.getElementsByClassName("box");

for(let i = 0; i < items.length; i++){
  items[i].style.background = "yellow";
}
HTMLCollection = Live collection It updates automatically when DOM changes.

3️⃣ getElementsByTagName()

Selects elements by their tag name (like div, p, h1). 

const paragraphs = document.getElementsByTagName("p");

Returns HTMLCollection (multiple elements).

4️⃣ querySelector() ⭐ (Modern & Most Used)

This method selects the first matching element using CSS selector syntax. 

const title = document.querySelector("#title");
const box = document.querySelector(".box");
const btn = document.querySelector("button");

✔ Uses CSS selector syntax ✔ Cleaner code ✔ Most recommended in modern development

5️⃣ querySelectorAll()

Selects all matching elements using CSS selector. Returns a NodeList. 

const allBoxes = document.querySelectorAll(".box");

allBoxes.forEach(function(element){
  element.style.color = "green";
});
NodeList supports forEach() HTMLCollection does NOT support forEach() directly.

🔎 Advanced Selection Examples

// Select input by type
document.querySelector("input[type='email']");

// Select child element
document.querySelector(".card h2");

// Select direct child
document.querySelector("ul > li");

3. Event Handling (Complete Guide)

JavaScript becomes powerful when it reacts to user actions. These actions are called events.

Event = Something that happens Event Listener = Code that waits and responds

🔹 What is an Event?

An event is an action performed by the user or browser.

  • Clicking a button
  • Typing in input field
  • Submitting a form
  • Hovering over element
  • Scrolling page

🔹 addEventListener() (Modern Method)

The best way to handle events in modern JavaScript. 

const button = document.querySelector("#btn");

button.addEventListener("click", function(){
  alert("Button Clicked!");
});

Syntax: 

element.addEventListener("eventName", functionName);

🔹 Common Event Types

Mouse Events

click, dblclick, mouseover

Keyboard Events

keydown, keyup

Form Events

submit, change

🔹 Example: Changing Text on Click

const heading = document.querySelector("#heading");

heading.addEventListener("click", function(){
  heading.textContent = "Text Changed!";
});

🔹 Using Named Functions

Instead of anonymous functions, you can define reusable functions. 

function showMessage(){
  console.log("Button clicked!");
}

button.addEventListener("click", showMessage);

🔹 Event Object

When an event occurs, JavaScript passes an event object. 

button.addEventListener("click", function(event){
  console.log(event);
});

Useful properties:

  • event.target
  • event.type
  • event.preventDefault()

🔹 Prevent Default Behavior

Used mostly in forms. 

form.addEventListener("submit", function(e){
  e.preventDefault();
  alert("Form submission prevented!");
});
Prevent Default = Stop browser’s automatic behavior

🔹 Real World Example: Dark Mode Toggle

const btn = document.querySelector("#darkBtn");

btn.addEventListener("click", function(){
  document.body.classList.toggle("dark-mode");
});

This is how interactive websites are built.

  • Always select element before adding event
  • Use addEventListener instead of inline JS

4. Form Validation (Complete Guide)

Form validation ensures that users enter correct and required data before submitting a form to the server.

Validation = Checking user input before processing it.

🔹 Why Form Validation is Important?

  • Prevents empty submissions
  • Improves user experience
  • Prevents invalid data
  • Enhances security

🔹 Basic Example: Required Field

const email = document.querySelector("#email").value;

if(email === ""){
  alert("Email is required!");
}

This checks if the user has entered any value.

🔹 Prevent Form Submission

By default, forms reload the page after submission. We use preventDefault() to stop this behavior. 

const form = document.querySelector("form");

form.addEventListener("submit", function(e){
  e.preventDefault();

  const email = document.querySelector("#email").value;

  if(email === ""){
    alert("Email required!");
  }
});
preventDefault() = Stop the browser from auto-submitting the form.

🔹 Email Format Validation

const email = document.querySelector("#email").value;

if(!email.includes("@")){
  alert("Enter valid email address");
}

This ensures email contains “@”.

🔹 Password Length Validation

const password = document.querySelector("#password").value;

if(password.length < 6){
  alert("Password must be at least 6 characters");
}

Always check password strength in real applications.

🔹 Real World Example: Complete Validation

const form = document.querySelector("#signupForm");

form.addEventListener("submit", function(e){
  e.preventDefault();

  const email = document.querySelector("#email").value;
  const password = document.querySelector("#password").value;

  if(email === "" || password === ""){
    alert("All fields are required");
    return;
  }

  if(password.length < 6){
    alert("Password must be at least 6 characters");
    return;
  }

  alert("Form submitted successfully!");
});

🔹 Types of Validation

Client-Side

Validation in browser using JavaScript

Server-Side

Validation in backend (Node, PHP, etc.)

HTML Built-in

Using required, minlength attributes

🔹 HTML Built-in Validation

Browsers automatically validate these attributes.

5. LocalStorage (Complete Guide)

LocalStorage is a Web Storage API that allows you to store data directly inside the user's browser. The stored data remains even after the browser is closed and reopened.

LocalStorage = Persistent storage inside the browser (5–10MB capacity)

🔹 Why LocalStorage is Important?

  • Stores data without backend
  • Data persists after page refresh
  • Improves user experience
  • Reduces unnecessary API calls

🔹 Basic Syntax

// Store data
localStorage.setItem("name", "Nikhil");

// Get data
const value = localStorage.getItem("name");

// Remove single item
localStorage.removeItem("name");

// Clear all storage
localStorage.clear();
setItem() = Save data getItem() = Read data removeItem() = Delete specific data clear() = Delete everything

🔹 Important Rule

LocalStorage stores data only in STRING format.

If you store objects or arrays, you must convert them using JSON.

🔹 Storing Objects (Using JSON)

const user = {
  name: "Nikhil",
  age: 22
};

// Convert object to string
localStorage.setItem("user", JSON.stringify(user));

// Convert back to object
const storedUser = JSON.parse(localStorage.getItem("user"));

console.log(storedUser.name);

JSON.stringify() converts object → string JSON.parse() converts string → object

🔹 Real World Example: Dark Mode Toggle

const toggleBtn = document.querySelector("#themeBtn");

toggleBtn.addEventListener("click", function(){
  document.body.classList.toggle("dark");

  if(document.body.classList.contains("dark")){
    localStorage.setItem("theme", "dark");
  } else {
    localStorage.setItem("theme", "light");
  }
});

// Load saved theme on page load
window.addEventListener("load", function(){
  const savedTheme = localStorage.getItem("theme");

  if(savedTheme === "dark"){
    document.body.classList.add("dark");
  }
});

This remembers user theme preference even after refresh.

🔹 Real World Example: Save Form Data

const input = document.querySelector("#username");

// Save input
input.addEventListener("input", function(){
  localStorage.setItem("username", input.value);
});

// Load saved input
window.addEventListener("load", function(){
  const savedName = localStorage.getItem("username");
  if(savedName){
    input.value = savedName;
  }
});

🔹 LocalStorage vs SessionStorage

LocalStorage

Data stays even after browser closes

SessionStorage

Data removed when tab closes

Cookies

Small data sent to server with requests

🔹 Where LocalStorage is Used?

  • Saving login tokens (small apps)
  • Remembering theme preference
  • Storing cart items
  • Saving user settings
  • Temporary offline data

🔹 Limitations

  • Only stores strings
  • Storage limit around 5MB
  • Not secure for sensitive data
  • Accessible via browser DevTools

🔹 Security Warning ⚠

Never store passwords or sensitive user data in LocalStorage. It can be accessed using browser developer tools.
